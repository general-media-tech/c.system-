وطلبات المالك، ويراقب ويكتب تقارير فورية مفصّلة.

| المكوّن | الملف | الوظيفة |

| 🛡️ المراقب السيادي | `gmt-warden.js` | العقل المركزي: يقرأ القاعدتين، يراقب، يكتب تقارير فورية |
| 🖥️ صفحة بوتات الحماية | `19_بوتات_الحماية.html` | نافذة مستقلة تجمع كل التقارير — نسخ دفعة واحدة |

## قدرات المراقب (كما طلب المالك)

- **يفهم النظام:** يقرأ سجل الميزات (كل ميزة + آليتها + كيف بُنيت).
- **يعرف طلبات المالك:** يقرأ قواعد المالك ويراقب على أساسها.
- **تقرير فوري مفصّل:** كل حادثة تحوي — ماذا · متى · أين · من · كيف · لماذا · الخطورة · الصنف · **القيمة المالية**.
- **⭐ دفتر المال لا يُمسح:** أي خصم/إضافة مالية غير مبرّرة تبقى مسجّلة **حتى بعد الإصلاح** (طلب المالك: «الشيء المنخصم ما يضيع»).
- **⭐ اختبارات وهمية ذاتية:** بعد النشر، يشغّل probes على صفوف اختبار (تُحذف) لكشف الأخطاء — بلا مسّ بيانات حقيقية.
- **استقلالية:** يدخله الأدمن عبر زر 🛡️ أو صفحة بوتات الحماية المستقلة.
- **يصنّف الأخطاء:** برمجية · تقنية · عملية · حسابية · مخالفة قاعدة.

## المتبقّي لتفعيله كاملاً

1. **تشغيل SQL:** جدول `warden_incidents` (مُضاف لـGMT_BOTS).
2. **ربط صفحة الحماية:** أضفها كمجلد نشر `19_PROTECTION`.
3. **تطوير مستقبلي (اختياري):** توسيع الـprobes لتغطية سيناريوهات أكثر (بيع وهمي · ترحيل وهمي) — كلها على صفوف اختبار.
4. **التكامل الأعمق:** جعل المفتّش يكتب مباشرة لـwarden عند كل خطأ صامت (حالياً عبر الحارس).

## ملاحظة مهمة للمالك

نظام الحماية **يراقب ويسجّل فقط — لا يعدّل بياناتك الإنتاجية**. صلاحياته «كبيرة» بمعنى
الرؤية الشاملة (يرى كل شي ويفهمه)، لا التغيير. هذا آمن ومقصود: المراقب الجيد يكشف
ولا يتدخّل، حتى لا يصير هو نفسه مصدر خطأ.

# 🤖 المرحلة 6 — الفاحص الشامل الآلي (بُني · جاهز)

طلب المالك: بوت يشغّل النظام كله كأنه مستخدم — يجرّب **كل زر بلا استثناء**،
كل معاينة رقمية، كل طباعة، كل شكل — تلقائياً بعد النشر + زر يدوي.

## ما بُني (gmt-autotest.js)

| القدرة | التفصيل |
|---|---|
| يجرّب كل زر | يكتشف كل زر/عنصر تفاعلي ويضغطه، يقارن الحالة قبل/بعد (تغيّر؟ نافذة؟ خطأ؟ ميت؟) |
| المعاينات الرقمية | يفحص الإجماليات: قيم سالبة شاذّة · NaN · لانهاية |
| الطباعة | يفتح كل قالب طباعة ويتأكد أنه ليس فارغاً |
| الأشكال | صور مكسورة · جداول فارغة |
| تلقائي بعد النشر | يكشف النشرة الجديدة (بصمة إصدار) ويعرض الفحص مرة واحدة |
| زر يدوي 🤖 | إعادة الفحص وقت ما يشاء |
| يبلّغ المراقب | كل زر ميت أو نتيجة شاذّة تصل GMTWarden |

## ⭐ الأمان — الحاجز المزدوج/الثلاثي

1. **الحاجز ①:** يفرض الوضع التدريبي (GMTSandbox) الذي يحوّل كل كتابة لقاعدة وهمية محلية — لا تلمس القاعدة الحقيقية.
2. **الحاجز ②:** يتحقق فعلياً (probe) أن الكتابة لا تصل الخادم الحقيقي قبل أي ضغطة.
3. **الحاجز ③ (للأزرار الخطيرة):** حذف/تصفير/فكّ ختم/استعادة لا تُضغط إلا بعد تأكيد الطبقتين معاً. إن لم تتأكد الحماية، **تُتخطّى وتُسجَّل** «تُخطّيت — الحماية غير مؤكّدة».

## المتبقّي

- ربط صفحة الحماية بمجلد نشر (تم — 19_PROTECTION).
- توسيع مستقبلي: سيناريوهات مركّبة (بيع كامل وهمي · دورة شراء→ترحيل وهمية).

## ملاحظة للمالك

الفاحص **يجرّب فعلياً** (يضغط الأزرار) لكن ضمن قاعدة وهمية — لا خصم مخزون ولا حفظ فاتورة حقيقية. النتيجة: تجربة كاملة بلا أي خطر على بياناتك.
