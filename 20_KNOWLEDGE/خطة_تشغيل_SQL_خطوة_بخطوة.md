## قبل أن تبدأ — ماذا تحتاج؟

النظام يستعمل قواعد بيانات على منصّة اسمها **Supabase**:
- **القاعدة الرئيسية (DB1)** — المنتجات · الفواتير · المبيعات · كل شيء أساسي.
- **قاعدة الكفالة (DB3)** — الكفالات والعقود.
- **قاعدة الموقع (DB4)** — محتوى الموقع والوكلاء.

لكل قاعدة، افتح صفحتها على Supabase → **SQL Editor** → وشغّل ملفاتها **بالترتيب المكتوب تحت، واحداً واحداً**.

> **كيف تشغّل ملفاً؟**
> ١. افتح الملف بمحرّر نصوص (Notepad) وانسخ كل محتواه.
> ٢. في Supabase: القائمة اليسرى → **SQL Editor** → **New query**.
> ٣. الصق → اضغط **Run** (أو Ctrl+Enter).
> ٤. انتظر **Success** ثم انتقل للملف التالي.

## 🔑 خطوة أخيرة: عيّن نفسك «أدمن»

بعد الانتهاء من القاعدة الرئيسية وتسجيل دخولك أول مرة، شغّل هذا في SQL Editor للقاعدة الرئيسية (بدّل `حسابك` باسم المستخدم):

```sql
update gmt_users set role='admin' where username='حسابك';
```

بدونها لن ترى أدوات الأدمن ولا البوتات.

## ✅ كيف تتأكد أن كل شيء تمام؟

بعد كل ملف: يجب أن تظهر **Success**. إن ظهر خطأ:
- **«already exists»** → طبيعي (شُغّل سابقاً). تجاهله وأكمل.
- **أي خطأ آخر** (مثل «column does not exist») → **توقّف** وأرسل نصّ الخطأ. لا تكمل.

بعد كل الملفات: افتح النظام كأدمن → زر **🧪 الفاحص الذاتي** (أسفل اليسار) → يجب أن يكون **أخضر**.
